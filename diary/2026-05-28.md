## What was done

- Started building Ilm AI from scratch
- First attempt had a poor project structure — everything was mixed together
- Decided to restart with a cleaner architecture: separated frontend and backend into two repositories
- Built the backend with FastAPI: auth (signup/login), file upload, RAG chat
- Built the frontend with plain HTML/CSS/JS: login, signup, dashboard pages
- The new structure is much better — frontend and backend are independent and easier to maintain
- Integrated Gemini API for embeddings (gemini-embedding-001) and chat (gemini-2.5-flash)
- Tested the full flow: signup → login → upload jupiter.pdf → ask questions → got correct answers

## Problems encountered

- First project structure was messy — mixed frontend and backend in one place
- ChromaDB did not work with Python 3.14 — incompatible
- OpenAI API quota was exhausted
- Gemini embedding model names were wrong — had to find correct ones by listing available models

## Solutions

- Restarted with clean separated structure: ilm-ai-backend and ilm-ai-frontend repos
- Replaced ChromaDB with numpy-based cosine similarity vector search
- Switched from OpenAI to Gemini API (free tier)
- Listed available Gemini models via API to find correct model names
